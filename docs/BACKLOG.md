# Публичный бэклог продукта: мультиарендная платформа записи для психологов

## 0) Видение и основные принципы
- Общая платформа, где любой психолог создаёт личный кабинет (арендатор = `Practitioner`).
- Публичная страница записи по `slug` и единый виджет. Внешний вид унифицированный, но с базовым брендированием.
- Клиент видит страницу психолога (инфо + кнопка «Записаться» c иконкой Telegram), форма записи отображается после входа через Telegram (логин/телефон подставляются автоматически), остаётся выбрать только дату/время.
- 7-дневный бесплатный триал без «шума» для пользователя. По окончании — сообщение с предложением оформить помесячную подписку. Данные и настройки сохраняются.
- Супер-админ (владелец, Telegram: @virtualsevt) видит сводную аналитику по всем кабинетам.

## 1) Архитектура и мультиарендность
- [ ] Tenant = `Practitioner`. Жёсткая фильтрация по `practitionerId` во всех моделях и запросах.
- [ ] Заголовки `x-practitioner-id` / `x-practitioner-slug` (уже частично реализовано в `frontend/src/api.js` и `backend/middleware/practitionerScope.js`).
- [ ] Роли: `super_admin`, `admin` (практик), `client`.
- [ ] Схема JWT: включать `role` и `practitionerId` (для админов).

## 2) Онбординг администратора (практика)
- [ ] Поток входа администратора через Telegram-бота (вариант A) или VK (вариант B — требует подтверждения).
- [ ] Deep-link → подтверждение личности → выбор/генерация `slug` → создание `Practitioner` и `User(role=admin)` → выдача JWT → редирект в админку.
- [ ] Поля у `Practitioner`: `trialStart`, `trialEnd`, `subscriptionStatus`, `planId`, `timezone`, `brandConfig` (json).
- [ ] Старт триала автоматически при создании практика. Никаких уведомлений в течение триала.

## 3) Админка (панель практика)
- [ ] Раздел «Профиль»: имя/описание/аватар, timezone, брендовые цвета, ссылки.
- [ ] Раздел «Расписание»: настройки (`ScheduleSetting`) + генерация слотов + ручное создание слота.
- [ ] Раздел «Бронирования»: список, статусы, перенос, отмена, экспорт.
- [ ] Раздел «Оплата и Подписка»: статус триала/подписки, кнопка «Оплатить», история платежей.
- [ ] Раздел «Интеграции»: Telegram/VK, вебхуки, уведомления.
- [ ] Guard на изменения (`billingGuard`): после триала изменения заблокированы до оплаты (чтение — доступно).

## 4) Публичная часть и виджет
- [ ] SPA-роутинг: `/p/:slug` (витрина психолога: инфо + кнопка «Записаться» с иконкой Telegram).
- [ ] При клике «Записаться»: запрашиваем вход через Telegram; после входа показываем форму записи с предзаполненными login/phone.
- [ ] Витрина слотов: получение слотов, выбор даты/времени, бронирование.
- [ ] Виджет для встраивания (iframe/script) с базовым конфигом цвета/логотипа/текстов.
- [ ] SEO/метатеги для публичной страницы, CORS для виджета.

## 5) Биллинг и триал
- [ ] Модели: `Plan`, `Subscription`, `Invoice`. У `Practitioner`: `trialStart`, `trialEnd`, `subscriptionStatus`, `planId`.
- [ ] Провайдер: Stripe или YooKassa (подтвердить). Сессия оплаты, вебхуки, статусы.
- [ ] Логика триала: в течение 7 дней — полный доступ. По окончании — показываем в админке «Оформите подписку» и ограничиваем изменения.
- [ ] Миграции, индексы по `practitionerId`.

## 6) Telegram/VK авторизация
- [ ] Клиент: подтверждение через Telegram-бота (получаем username и телефон через «Поделиться номером»), выдача client JWT. Форма записи требует client JWT и `tgPhone`.
- [ ] Админ: отдельный поток через Telegram-бота (или VK — подтвердить выбор). Код (`TgAuthCode`) помечается `type: 'admin'|'client'`.
- [ ] Настройки уведомлений: напоминания о сессиях, оповещения об отмене/переносе.

## 7) Супер-админ (@virtualsevt)
- [ ] Роль `super_admin`, доступ к сводной панели: число активных практиков, новых за период, статусы подписок, загрузка слотов, конверсии в оплату.
- [ ] Просмотр и отключение практиков (soft disable), аудит логов, фильтры.
- [ ] Техническая поддержка: доступ к карточке практика, сброс токенов, перевыпуск ссылок.

## 8) Безопасность и приватность
- [ ] Строгий scoping по `practitionerId` во всех сервисах и запросах.
- [ ] Rate limits на публичные эндпоинты, защита вебхуков, проверка подписей.
- [ ] Логи: мутаторы/ошибки/вебхуки, маскирование телефон/почты в логах.
- [ ] RBAC: разграничение ролей (`super_admin`/`admin`/`client`).
